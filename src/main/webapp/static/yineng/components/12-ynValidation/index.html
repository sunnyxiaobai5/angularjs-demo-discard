API
为空判断
  • 必要参数:required
邮箱格式验证
  • 必要参数:type="email"
长度验证
  • 必填参数:ng-maxlength
  • 可选参数:ng-minlength
ynValidation可配置参数
  • type(错误信息显示位置)
    • 参数1:block（默认参数,将提示信息显示在下方）
    • 参数2:inline(将提示信息显示在右边)
    • yn-validation="{type:'inline'}"
  • custom(自定义逻辑配置)
    • 参数:传入的参数需要个{content:'错误提示':fun(){}}这样的对象
    • fun()方法说明:必须返回true或者false,比如验证正常，返回true,错误返回false
    • 详细用户参照下面用例
  • tipHead(为空判读无法获取头名字信息时，自定义传入)
    • 参数:自定义名字
    • yn-validation="{tipHead:'名字'}"
  • inputGroup(是否是inputGroup形式的输入框，参照下面用例)
    • 参数:true(不传默认不是inputGroup)
    • yn-validation="{inputGroup:true}"
  • isClick(组织机构选中，时间组件这种时需要加的参数)
    • 参数:true
    • yn-validation="{isClick:true}"
    • 用例中未写此参数的DEMO
关于点击提交时的全局验证
  • 提交按钮必须在form标签中，并加上属性subButt
  • 如果是ynModel模态弹出框，只需在弹出框中配置有提交按钮onfocus:true（规范要求提交按钮必须配）
